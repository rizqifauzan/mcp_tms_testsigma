How it works

From clone to "list my test cases" in under 5 minutes.

1

Deploy the server

Fork the repo, push to Vercel. Region sin1 recommended. No DB required.

vercel --prod
2

Grab your API key

In Testsigma TMS, open Settings → API Keys and generate a personal token. Copy it.

FAQ

Questions, answered.

What's the difference between Testsigma TMS and the Testsigma automation platform?

TMS is Testsigma's Test Management product (manual test cases, plans, results). The automation platform is a separate product with its own API. This MCP server targets TMS only.

Do I need to host this myself?

Yes — but it takes one click on Vercel and runs comfortably on the free tier. Self-hosting also means your API key never goes through a third-party server.

Which Claude clients are supported?

Any MCP-compatible client. Tested primarily against Claude Code (CLI, desktop, and web). Claude Desktop also works via a custom HTTP MCP entry.

Is Jira integration supported?

Phase 3 will provide orchestration tools that combine Testsigma data with Jira (via your separate Jira MCP). Direct Jira-link writes inside TMS aren't exposed by the TMS API.

Is it safe to give it write access to my test suite?

Every write tool requires explicit user approval in Claude Code. The Phase 2 release was verified end-to-end with net-zero state changes before merge.
