Setup

Connect in 5 minutes — Terminal or Browser.

Two auth modes are supported: header-based for Claude Code (most secure) and URL-path-based for Claude Web (when custom headers aren't available).

1

Generate your Testsigma API key

In Testsigma TMS web app, open your profile menu (top-right) → Settings → API Keys → Generate New Key. Name it something traceable like Personal MCP — <your name>.

⚠️ Important: Every team member generates their own key — do not share. This preserves audit trail and per-user permissions.

Copy the key — it's a long JWT starting with eyJ... (~350 chars). You won't be able to see it again.

2A

Claude Code (Terminal)

Most secure

Uses the X-Testsigma-Key HTTP header — your key stays out of URLs and access logs.

Add the MCP server

claude mcp add tms https://mcp-tms-testsigma.vercel.app/mcp \
  --transport http \
  --header "X-Testsigma-Key: <YOUR_KEY>"

Verify

claude mcp list
# Should show: tms · http · 25 tools

Start a session

claude
# Inside Claude Code:
/mcp
# Verify "tms" appears with 25 tools
If you see error: unknown option '--url': URL is positional, not a flag. Drop --url and pass the URL directly after the server name.
2B

Claude Web (claude.ai)

Browser-friendly

Claude Web doesn't support custom headers, so the key goes in the URL path. Treat the URL itself like a password.

Open connector settings

  1. Go to claude.ai
  2. Click your avatar (bottom-left) → Settings
  3. Open Connectors → Add custom connector

Paste this URL

# Replace <YOUR_KEY> with your TMS API key (eyJ...)
https://mcp-tms-testsigma.vercel.app/mcp/<YOUR_KEY>

Verify

In a new chat, click the 🔌 connector icon below the input. You should see Testsigma TMS with 25 tools.

⚠️ Security: The URL contains your TMS key — don't share it in public Slack, commit to git, or screenshot it. If it leaks, immediately revoke the key in TMS Settings → API Keys.
3

Try your first prompts

Copy any of these into Claude — it will pick the right tool automatically.

Read

"List 5 folders in project Grow"

Create

"Create a test case in folder Checkout, title 'Verify guest checkout', priority Major"

Bulk update

"Reassign all draft TCs in folder X to john@cata.com"

Execute

"Start a static test run from TC GR-46, GR-47, GR-48 and mark GR-46 as Passed"

Troubleshooting

I see 0 tools in /mcp or "Failed to connect"
  1. Verify the API key is still ACTIVE in TMS Settings → API Keys.
  2. Restart Claude Code (/exit, then claude) to refresh tool cache.
  3. Re-add the server: claude mcp remove tms then add again with the latest URL.
  4. Test the endpoint directly with curl (see README).
How do I rotate or revoke my API key?

In TMS, open Settings → API Keys → click Revoke next to the old key, then Generate New Key. Update the URL or header in your client. Old key stops working immediately.

A delete tool was called without my approval

Destructive tools (delete_test_case, delete_folder, delete_test_plan, delete_test_run) require explicit confirmation in Claude. Check your Permissions settings — you may have previously chosen "Always allow" for that tool. You can reset per-tool permissions in Claude Code's settings.

FAQ

Questions, answered.

What's the difference between Testsigma TMS and the Testsigma automation platform?

TMS is Testsigma's Test Management product (manual test cases, plans, results). The automation platform is a separate product with its own API. This MCP server targets TMS only.

Do I need to host this myself?

Yes — but it takes one click on Vercel and runs comfortably on the free tier. Self-hosting also means your API key never goes through a third-party server.

Which Claude clients are supported?

Any MCP-compatible client. Two auth flows are supported: header-based (X-Testsigma-Key) for Claude Code CLI/desktop, and URL-path-based (/mcp/<key>) for Claude Web where custom headers aren't available. Both paths share the same 25 tools.

Is it safe to give it write access to my test suite?

Every destructive tool (delete_*) requires explicit approval in your Claude client. Phases 2 and 4 were each verified end-to-end with net-zero state changes (everything created during testing was deleted in the same run) before merge.

Should everyone on the team share one API key?

No — each team member should generate their own personal TMS API key. This preserves per-user audit trail in TMS, respects per-user permissions, and makes offboarding clean (revoke one key, everyone else is unaffected).
